# Система управления прогрессом заявок

## Обзор

Система динамического отображения прогресса заявок клиентов в личном кабинете. Менеджер может управлять текущим этапом заявки, а клиент видит актуальный статус на главной странице dashboard.

## Реализованные компоненты

### Backend

#### 1. Модель данных: `backend/models/application_progress.py`
- **ApplicationStage** - класс для представления одного этапа
- **ApplicationProgress** - основной класс для управления прогрессом
- **STAGES_BY_TYPE** - предопределенные этапы для разных типов заявок:
  - `vnj_investment` - ВНЖ по инвестициям (6 этапов)
  - `business_immigration` - Бизнес-иммиграция (6 этапов)
  - `family_reunification` - Семейное воссоединение (6 этапов)
  - `consultation` - Консультация (4 этапа)

#### 2. API эндпоинты: `backend/routes/application_progress.py`

**GET /api/application/progress**
- Получить прогресс текущего пользователя
- Требует авторизации
- Возвращает: stages, progress_percentage, current_stage_id

**GET /api/application/progress/<user_id>**
- Получить прогресс конкретного пользователя
- Требует права admin/manager
- Для просмотра прогресса клиентов

**PUT /api/application/progress/<user_id>**
- Обновить текущий этап пользователя
- Требует права admin/manager
- Body: `{"current_stage_id": "submission"}`

**GET /api/application/types**
- Получить список всех типов заявок и их этапов
- Требует авторизации

#### 3. База данных: `backend/models/user.py`
Добавлены новые поля в таблицу users:
- `application_type` - тип заявки (по умолчанию: 'vnj_investment')
- `current_stage` - текущий этап (по умолчанию: 'consultation')

### Frontend

#### 1. JavaScript: `frontend/js/progress-tracker.js`
- Автоматическая загрузка прогресса при открытии dashboard
- Динамическое построение таймлайна
- Поддержка русского и английского языков
- Визуальное выделение текущего этапа
- Функция `window.refreshProgressTracker()` для ручного обновления

#### 2. Интеграция: `frontend/lk/dashboard.html`
- Подключен скрипт `progress-tracker.js`
- Таймлайн автоматически обновляется при загрузке страницы

## Использование

### Для клиента

1. Войти в личный кабинет
2. Открыть главную страницу (Dashboard)
3. В секции "Application Timeline" отображается текущий прогресс
4. Активный этап выделен цветом и имеет метку "Текущий этап"
5. Завершенные этапы отмечены галочкой
6. Будущие этапы показаны серым цветом

### Для менеджера

#### Обновление этапа через API:

```bash
# Получить текущий прогресс клиента
curl -X GET http://localhost:5000/api/application/progress/1 \
  -H "Authorization: Bearer YOUR_TOKEN"

# Обновить этап клиента
curl -X PUT http://localhost:5000/api/application/progress/1 \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"current_stage_id": "submission"}'
```

#### Доступные этапы для ВНЖ по инвестициям:
1. `consultation` - Первичная консультация и стратегия
2. `documents` - Сбор документов
3. `legal_review` - Юридическая проверка и перевод
4. `submission` - Подача в министерство
5. `processing` - Обработка государственными органами
6. `approval` - Одобрение и получение ВНЖ

#### Доступные этапы для бизнес-иммиграции:
1. `consultation` - Бизнес-консультация
2. `company_setup` - Регистрация компании
3. `documents` - Подготовка документов
4. `submission` - Подача заявки
5. `processing` - Рассмотрение
6. `approval` - Одобрение

#### Доступные этапы для семейного воссоединения:
1. `consultation` - Консультация
2. `documents` - Сбор документов
3. `legalization` - Легализация документов
4. `submission` - Подача заявки
5. `processing` - Рассмотрение
6. `approval` - Одобрение

#### Доступные этапы для консультации:
1. `initial` - Первичная консультация
2. `analysis` - Анализ документов
3. `recommendations` - Рекомендации
4. `completed` - Завершено

### Изменение типа заявки

Тип заявки хранится в поле `application_type` таблицы users. Для изменения:

```sql
UPDATE users SET application_type = 'business_immigration' WHERE id = 1;
```

Или через API (требуется добавить соответствующий эндпоинт).

## Визуальные состояния этапов

### Завершенный этап
- Зеленая галочка в круге
- Полная непрозрачность
- Дата завершения (если доступна)

### Активный этап
- Пульсирующая точка в круге с оранжевой рамкой
- Выделенный фон
- Метка "Текущий этап" / "Current Stage"
- Описание действий

### Будущий этап
- Серая иконка часов
- Полупрозрачность (opacity: 50%)
- Описание этапа

## Технические детали

### Автоматическое обновление
- Прогресс загружается при каждом открытии dashboard
- Использует localStorage для кэширования профиля пользователя
- Поддерживает множественные API endpoints (fallback)

### Локализация
- Автоматическое определение языка из профиля пользователя
- Поддержка русского (ru) и английского (en)
- Все тексты хранятся в модели на обоих языках

### Безопасность
- Все API требуют JWT авторизации
- Обновление прогресса доступно только admin/manager
- Клиент может видеть только свой прогресс

## Расширение системы

### Добавление нового типа заявки

1. Добавить в `STAGES_BY_TYPE` в `backend/models/application_progress.py`:

```python
'new_type': [
    {
        'stage_id': 'stage1',
        'title_ru': 'Этап 1',
        'title_en': 'Stage 1',
        'description_ru': 'Описание',
        'description_en': 'Description',
        'order': 1
    },
    # ... другие этапы
]
```

2. Обновить `application_type` пользователя в БД

### Добавление уведомлений

Можно расширить систему, добавив:
- Email уведомления при смене этапа
- Push уведомления в браузере
- Webhook для интеграции с внешними системами

## Тестирование

1. Запустить backend: `cd backend && python app.py`
2. Запустить frontend: `python -m http.server 8080`
3. Войти в систему
4. Открыть http://localhost:8080/frontend/lk/dashboard.html
5. Проверить отображение таймлайна
6. Через API изменить этап и обновить страницу

## Troubleshooting

### Прогресс не отображается
- Проверить консоль браузера на ошибки
- Убедиться, что backend запущен
- Проверить наличие токена в localStorage
- Проверить, что у пользователя установлены application_type и current_stage

### Этап не обновляется
- Проверить права доступа (admin/manager)
- Убедиться, что stage_id корректный для данного типа заявки
- Проверить логи backend сервера

### Неправильный язык
- Проверить поле locale в профиле пользователя
- Очистить localStorage и войти заново
